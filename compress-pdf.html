Understanding Compression Levels

Low Compression

Low compression applies minimal optimization techniques to reduce file size by approximately 10-30% with virtually no visible quality loss. This level is ideal for documents where quality is paramount, such as legal contracts, architectural drawings, high-resolution presentations, and documents with embedded vector graphics. Low compression primarily removes redundant metadata and optimizes font embedding without affecting images or visual elements.

Medium Compression (Recommended)

Medium compression provides the best balance between file size reduction and quality preservation, typically reducing file size by 40-60%. This level uses intelligent image downsampling to 150 DPI for images (sufficient for screen viewing), removes duplicate resources, optimizes page content streams, and compresses fonts more efficiently. Medium compression is perfect for general business documents, reports, ebooks, and most everyday PDF files.

High Compression

High compression aggressively reduces file size by 70-85% for documents with many images, or 50-70% for text-heavy documents. This level uses more aggressive image compression (downsampling to 96 DPI), JPEG compression at 75% quality for photos, and may convert some images to grayscale. High compression is ideal for archiving large documents, reducing storage space, sending via email, or when file size is more important than absolute quality.

Technical Details: How Our PDF Compressor Works

Our PDF compressor utilizes the PDF-lib library with custom optimization algorithms. The compression process involves several techniques:

  • Image Optimization: Images are often the largest component of PDFs. Our compressor downsamples images to appropriate resolutions (96-150 DPI) and applies JPEG compression to photographic images while preserving PNG compression for graphics and text.
  • Font Optimization: Many PDFs embed entire font families when only a few characters are used. Our compressor subsets fonts to include only the characters actually used in the document.
  • Metadata Removal: Optional metadata like document properties, annotations, and form fields can be stripped to reduce size (available in higher compression levels).
  • Stream Compression: PDF content streams are compressed using Flate or LZW compression algorithms.
  • Duplicate Object Removal: Identical resources (images, fonts, patterns) used multiple times are stored once and referenced, reducing file size.

Best Practices for PDF Compression

Choose the Right Level for Your Needs

Start with medium compression and check the results. If the file is still too large, try high compression. If you notice quality issues, use low compression. Always keep your original file until you're satisfied with the compressed version.

When scanning documents, scan at appropriate resolutions (300 DPI for text, 150 DPI for screen viewing). Scanning at unnecessarily high resolutions creates large PDFs that even compression can't fully fix.

Troubleshooting Common Issues

File Still Too Large After Compression?

Some PDFs, particularly those with many high-resolution images or complex vector graphics, may still be large even after maximum compression. Consider:

  • Re-scanning original documents at lower resolution
  • Extracting and re-compressing images before re-embedding
  • Splitting the PDF into smaller documents
  • Converting to a different format (e.g., PDF/A might be larger)

Quality Loss Noticeable?

If you notice quality degradation, especially in images or text rendering, switch to a lower compression level. Low compression maintains near-original quality while still providing some size reduction. For text-only documents, compression has minimal visible impact.

PDF Contains Forms or Annotations?

Compressed PDFs retain form fields, annotations, and interactive elements. However, very high compression may affect form functionality. Use low or medium compression for interactive PDFs.

Understanding PDF Size Components

PDF files consist of several components that contribute to overall size:

  • Images (60-80% of size): Photographs, scans, and graphics are typically the largest component. Image compression provides the biggest size reduction.
  • Fonts (10-20%): Embedded font files, especially complete font families with many characters.
  • Text Content (5-10%): The actual text of the document, usually well-compressed already.
  • Metadata and Structure (5-10%): Document properties, bookmarks, annotations, and structural information.

Our compressor optimizes all these components proportionally based on your selected compression level.

Frequently Asked Questions

How much can I reduce PDF size?

Size reduction varies widely based on content. Text-only PDFs might compress 30-50%. PDFs with many high-resolution images can compress 70-90%. Scanned documents (which are essentially images) typically compress 50-80%.

Will compression affect text quality?

Text quality remains excellent even at high compression levels because text is stored as vectors, not images. Only scanned documents (text as images) may show quality loss at high compression.

Do you add watermarks to compressed PDFs?

Absolutely not. The compressed PDF contains only your original content with no added watermarks, logos, or promotional materials.

What's the maximum file size?

Since processing happens locally, limits depend on your device's available memory. Most modern devices can handle PDFs up to 500MB easily. For files larger than 200MB, we recommend using Chrome or Edge for optimal performance.

Is compression lossy or lossless?

Our compressor uses both lossy (for images at medium/high levels) and lossless (for text and fonts) compression. Low compression is mostly lossless. Choose lower levels if you need mathematically perfect reproduction.
